## PR title format (required or will fail)

PR titles must follow **Conventional Commits**:

`<type>: <short summary>`

Allowed `type` values:
`feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `ci`, `build`, `perf`, `style`, `revert`

Examples:
- `fix: handle empty staged file list in docstring gate`
- `chore: tighten CI to mirror pre-commit`
